## Database (`/database`)

### Tables
| Table | Primary Key | Purpose |
|-------|-------------|---------|
| `players` | `player_id` | Player names, career span |
| `teams` | `team_id` (3-char) | Team info, colors |
| `weeks` | `(season, week)` | Season type (REG/POST) |
| `player_week` | `(player_id, season, week)` | Individual game stats |
| `team_week` | `(team_id, season, week)` | Team game stats |

### Key Functions
- **`player_stats(player_id, season_start, season_end, week_start, week_end, season_type)`**
  - Returns 190+ aggregated columns (passing, rushing, receiving, totals, fantasy)
  - All params optional (NULL = no filter)

- **`team_stats(team_id, ...same params...)`**
  - Returns 230+ aggregated columns (offense + defense mirrors)

### Connection
- **Development:** SSH tunnel вҶ’ `localhost:5433`
- **Production:** Direct on VPS вҶ’ `localhost:5432`

---

## ETL Pipeline (`/etl`)

R-based pipeline using `nflreadr` package.

**Data Flow:** nflverse API вҶ’ Extract вҶ’ Transform вҶ’ Load вҶ’ PostgreSQL

**Coverage:** 1999-present, ~40 player metrics/week, ~60 team metrics/week

**Positions Tracked:** QB, WR, RB, TE

**Key Files:**
- `src/etl_pipeline.R` - Main orchestrator
- `src/transform_players.R` - Player stat calculations
- `src/transform_teams.R` - Team stat calculations

### API Endpoints
| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/players` | GET | List/search players (`?q=name&limit=100`) |
| `/api/players/[id]` | GET | Player stats with date filters |
| `/api/teams` | GET | List all 32 teams |
| `/api/teams/[id]` | GET | Team stats with date filters |

**Date Filter Params** (all endpoints with `[id]`):
- `seasonStart`, `seasonEnd` (1999-2030)
- `weekStart`, `weekEnd` (1-22)
- `seasonType` (`REG`, `POST`, `REG,POST`)

## Running Locally

```bash

cd database && ./scripts/ssh-tunnel.sh

# 2. Set up environment
cd stats-app
cp .env.example .env.local
# Edit .env.local with credentials

# 3. Install and run
npm install
npm run dev

# 4. Test
curl http://localhost:3000/api/teams
curl "http://localhost:3000/api/players?q=mahomes"
curl "http://localhost:3000/api/teams/KC?seasonStart=2024&seasonType=REG"
```

## Key Design Decisions

1. **`pg` over Prisma** - Database already has stored procedures; ORM adds unnecessary abstraction
2. **URL state for filters** - Shareable links, SSR-compatible
3. **No authentication** - Public tool, all features accessible
4. **Server Components** - Leverage Next.js 16 for performance
5. **Minimal design** - Clean, focused UI using frontend-design skill
