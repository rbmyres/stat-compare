# NFL Statistics Comparison Platform

A full-stack web application for analyzing and comparing NFL player and team statistics.

## Architecture Overview

```
┌─────────────────────────────────────────────────────────────────┐
│                         Frontend                                 │
│              Next.js 16 / React 19 / Tailwind v4                │
│                    /stats-app                                    │
└─────────────────────┬───────────────────────────────────────────┘
                      │ Server Components (direct DB queries)
                      ▼
┌─────────────────────────────────────────────────────────────────┐
│                      PostgreSQL                                  │
│          Hosted on VPS (AWS Lightsail)                          │
│   Access: SSH tunnel (dev) or direct localhost (prod)           │
└─────────────────────────────────────────────────────────────────┘
                      ▲
                      │ ETL Pipeline (R)
                      │
┌─────────────────────────────────────────────────────────────────┐
│                    nflverse API                                  │
│            Play-by-play data source                             │
└─────────────────────────────────────────────────────────────────┘
```

## Database (`/database`)

### Tables
| Table | Primary Key | Purpose |
|-------|-------------|---------|
| `players` | `player_id` | Player names, career span |
| `teams` | `team_id` (3-char) | Team info, colors |
| `weeks` | `(season, week)` | Season type (REG/POST) |
| `player_week` | `(player_id, season, week)` | Individual game stats (~50 cols) |
| `team_week` | `(team_id, season, week)` | Team game stats (~70 cols) |

### Key Functions
- **`player_stats(player_id, season_start, season_end, week_start, week_end, season_type)`**
  - Returns ~142 aggregated columns (passing, rushing, receiving, totals, fantasy)
  - All params optional (NULL = no filter)

- **`team_stats(team_id, ...same params...)`**
  - Returns ~252 aggregated columns (offense + defense mirrors)

### Connection
- **Development:** SSH tunnel → `localhost:5433`
- **Production:** Direct on VPS → `localhost:5432`

## ETL Pipeline (`/etl`)

R-based pipeline using `nflreadr` package.

**Data Flow:** nflverse API → Extract → Transform → Load → PostgreSQL

**Coverage:** 1999-present, ~40 player metrics/week, ~60 team metrics/week

**Positions Tracked:** QB, WR, RB, TE

**Key Files:**
- `src/etl_pipeline.R` - Main orchestrator
- `src/transform_players.R` - Player stat calculations
- `src/transform_teams.R` - Team stat calculations

### Player Listing Pages
- **Navbar dropdown**: "Players" shows sub-links (Passing, Rushing, Receiving, Totals); desktop hover, mobile tap-to-expand
- **Data**: Each page calls `player_stats(NULL, ...)` to fetch all players, filters by category (e.g., `pass_attempts > 0`)
- **Sortable table**: Click column headers to sort asc/desc; player name links to `/players/[id]`
- **Basic/Advanced toggle**: Passing, Rushing, Receiving pages switch between basic and advanced column sets; Totals has no toggle
- **Column definitions**: `lib/columns.ts` defines `ColumnDef` (`label`, `key`, `format`) arrays mirroring the 7 stat builder functions

### Team Listing Pages
- **Navbar dropdown**: "Teams" shows sub-links (Passing, Rushing, Total, Situational)
- **Data**: Each page calls `team_stats(NULL, ...)` to fetch all 32 teams
- **Offense/Defense toggle**: Switches between `off_` and `def_` prefixed stat columns
- **Basic/Advanced toggle**: Passing and Rushing pages also have Basic/Advanced; Total and Situational do not
- **Column definitions**: `lib/team-columns.ts` defines 12 `TeamColumnDef` arrays (6 off + 6 def)

### Filter System
- URL params: `startYear`, `startWeek`, `endYear`, `endWeek`, `seasonType`
- Defaults: 2025, Wk 1–22, REG (cleared from URL when default)
- Shared parsers between client (`useQueryStates`) and server (`searchParamsCache`)
- `shallow: false` triggers server re-render on filter change

## Key Design Decisions

1. **`pg` over Prisma** - Database already has stored procedures; ORM adds unnecessary abstraction
2. **URL state for filters** - Shareable links, SSR-compatible via nuqs
3. **No authentication** - Public tool, all features accessible
4. **Server Components** - Detail pages query DB directly, no API layer needed
5. **Minimal design** - Clean, focused UI; NFL Navy/Red theme
6. **Horizontal scrollable tables** - Fits many stats without vertical bloat
7. **Position-aware display** - Shows relevant stat sections first based on player position
8. **Column-driven tables** - `ColumnDef` arrays (label + key + format fn) power sortable leaderboards; columns imported client-side to avoid serializing functions across server/client boundary
