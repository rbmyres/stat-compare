# Database Summary

NFL statistics database using PostgreSQL via Supabase.

## Schema Overview

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   players   │     │    teams    │     │    weeks    │
│─────────────│     │─────────────│     │─────────────│
│ player_id   │◄────│ team_id     │     │ season (PK) │
│ first_name  │     │ abbr        │     │ week (PK)   │
│ last_name   │     │ display_name│     │ season_type │
│ first_season│     │ nickname    │     └──────┬──────┘
│ last_season │     │ primary_color│           │
└──────┬──────┘     └──────┬──────┘            │
       │                   │                   │
       │    ┌──────────────┴───────────────────┘
       │    │              │
       ▼    ▼              ▼
┌──────────────────────────────────────────────┐
│                 player_week                   │
│──────────────────────────────────────────────│
│ PK: (player_id, season, week)                │
│ FK: player_id → players                      │
│ FK: team_id → teams                          │
│ FK: (season, week) → weeks                   │
│                                              │
│ Columns: ~50 (passing, rushing, receiving,   │
│          totals, EPA/WPA metrics)            │
└──────────────────────────────────────────────┘

┌──────────────────────────────────────────────┐
│                  team_week                    │
│──────────────────────────────────────────────│
│ PK: (team_id, season, week)                  │
│ FK: team_id → teams                          │
│ FK: (season, week) → weeks                   │
│                                              │
│ Columns: ~70 (offense + defense stats)       │
│ Generated: 9 computed columns (STORED)       │
└──────────────────────────────────────────────┘
```

## Tables

### Reference Tables
| Table | PK | Purpose |
|-------|-----|---------|
| `players` | `player_id` | Player names, career span |
| `teams` | `team_id` (3-char) | Team names, colors |
| `weeks` | `(season, week)` | Season type (REG/POST) |

### Fact Tables
| Table | PK | Stats |
|-------|-----|-------|
| `player_week` | `(player_id, season, week)` | Individual game stats |
| `team_week` | `(team_id, season, week)` | Team game stats (off/def) |

### Generated Columns (team_week)
Automatically computed on insert:
- `off_plays_total`, `off_yards_total`, `off_first_downs`, `off_touchdowns`
- `off_epa`, `off_wpa`, `off_success_total`, `off_explosive_plays`
- Same 8 columns for defense (`def_*`)

## Indexes

### player_week
| Index | Columns | Notes |
|-------|---------|-------|
| `idx_pw_player_id` | `player_id` | Single-player lookups |
| `idx_pw_player_season_week` | `player_id, season, week` | Player history |

| `idx_pw_season_week_position` | `season, week, position` | Leaderboards |

| Index | Columns | Notes |

| `idx_tw_season_covering` | INCLUDE clause | Key stats |

## Query Functions

### `player_stats(player_id, season_start, season_end, week_start, week_end, season_type)`
Aggregates player statistics across a date range.

**Returns:** 190+ columns including:
- Raw totals (pass_yards, rush_yards, rec_yards, etc.)
- Calculated rates (completion %, yards per attempt, EPA per play)
- Per-game averages
- Passer rating, fantasy points (PPR)

**Example:**
```sql
SELECT * FROM player_stats('00-0036442', 2024, 2024, 1, 18, 'REG');
```

### `team_stats(team_id, season_start, season_end, week_start, week_end, season_type)`
Aggregates team offensive and defensive stats.

**Returns:** 230+ columns including:
- Offense: passing, rushing, total, situational (3rd/4th down)
- Defense: same categories as offense
- Per-game, per-drive, per-play rates

**Example:**
```sql
SELECT * FROM team_stats('BAL', 2024, 2024, NULL, NULL, 'REG');
```

### Parameter Notes
- All parameters are optional (NULL = no filter)
- `season_type`: Comma-separated, validated (`'REG'`, `'POST'`, or `'REG,POST'`)
- Invalid season_type raises error

## Row Level Security (RLS)

| Role | SELECT | INSERT/UPDATE/DELETE |
|------|--------|---------------------|
| `anon` | All tables | None |
| `authenticated` | All tables | None |
| `service_role` | All tables | All tables |

**Note:** ETL must connect with `service_role` key, not `anon` key.

## ETL Pipeline

Located in `/etl/`. Written in R using `nflreadr` and `RPostgres`.

### Data Flow
```
nflverse API → Extract → Transform → Load → PostgreSQL
```

### Key Files
| File | Purpose |
|------|---------|
| `config/database.R` | Connection management |
| `src/extract.R` | Download from nflverse |
| `src/transform_players.R` | Calculate player stats |
| `src/transform_teams.R` | Calculate team stats |
| `src/load.R` | Insert into database |

### Load Functions
```r
load_player_data(player_data, season)  # → players table
load_team_data(team_data)              # → teams table
load_weeks_data(schedule_data)         # → weeks table
load_player_stats(player_stats)        # → player_week table
load_team_stats(team_stats)            # → team_week table
clear_season_data(season)              # Delete season for re-run
```

### Environment Variables
```
NFL_DB_NAME, NFL_DB_HOST, NFL_DB_PORT
NFL_DB_USER, NFL_DB_PASSWORD, NFL_DB_SSLMODE
```

## Key Design Decisions

1. **Composite PKs** over surrogate keys - natural keys (player_id, season, week) are meaningful
2. **Generated columns** for derived totals - always consistent, no recalculation
3. **Partial indexes** for recent seasons - 80% of queries hit recent data
4. **COALESCE in functions** - prevents NULL propagation in aggregations
5. **Boolean win/loss/tie** columns - faster filtering than string comparisons
6. **Separate off/def columns** in team_week - enables direct offense vs defense queries

## Common Queries

```sql
-- Season leaders (passing yards)
SELECT * FROM player_stats(NULL, 2024, 2024, 1, 18, 'REG')
WHERE pass_attempts > 100
ORDER BY pass_yards DESC;

-- Team comparison
SELECT team_id, off_yards_per_game, def_yards_per_game
FROM team_stats(NULL, 2024, 2024, NULL, NULL, 'REG')
ORDER BY off_yards_per_game DESC;

-- Player career stats
SELECT * FROM player_stats('00-0036442', NULL, NULL, NULL, NULL, 'REG,POST');
```
