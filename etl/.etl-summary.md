# NFL ETL Pipeline Summary

## Overview

An R-based ETL pipeline that extracts NFL play-by-play data from **nflverse**, transforms it into player and team weekly statistics, and loads it into a PostgreSQL database.

```
nflverse API (nflreadr)
         │
         ▼
┌─────────────────┐
│  EXTRACT        │ → pbp_data, player_data, team_data, schedule_data
└────────┬────────┘
         ▼
┌─────────────────┐
│  TRANSFORM      │ → player_stats (40+ metrics), team_stats (60+ metrics)
└────────┬────────┘
         ▼
┌─────────────────┐
│  ENHANCE        │ → Add team/position/win-loss metadata
└────────┬────────┘
         ▼
┌─────────────────┐
│  LOAD           │ → PostgreSQL: players, teams, weeks, player_week, team_week
└─────────────────┘
```

## File Structure

| File | Purpose |
|------|---------|
| `src/etl_pipeline.R` | Main orchestrator - runs Extract → Transform → Enhance → Load |
| `src/extract.R` | Downloads data from nflverse API |
| `src/transform_players.R` | Calculates player statistics (passing/rushing/receiving) |
| `src/transform_teams.R` | Calculates team statistics (offense/defense) |
| `src/load.R` | Inserts data into PostgreSQL |
| `src/utils.R` | Helpers: logging, progress tracking, retry logic, data enhancement |
| `config/database.R` | Database connection management |
| `scripts/run_2024.R` | Run ETL for 2024 season only |
| `scripts/run_historical.R` | Backfill all seasons (1999-current) |
| `setup.R` | Install required R packages |

## How to Run

### Prerequisites
1. Install R packages: `Rscript setup.R`
2. Create `.env` file with database credentials:
   ```
   NFL_DB_NAME=your_db_name
   NFL_DB_HOST=your_host
   NFL_DB_PORT=5432
   NFL_DB_USER=your_user
   NFL_DB_PASSWORD=your_password
   NFL_DB_SSLMODE=require
   ```

### Run Commands
```bash
cd StatsApp/etl

# Single season (clears existing 2024 data first)
Rscript scripts/run_2024.R

# All seasons (1999-current, preserves existing data)
Rscript scripts/run_historical.R

# Custom seasons via command line
Rscript src/etl_pipeline.R 2022 2023 2024
```

## Pipeline Phases

### 1. Extract (`extract.R`)

Downloads four datasets from nflverse using `nflreadr`:

| Function | Data | Description |
|----------|------|-------------|
| `extract_pbp_data(seasons)` | Play-by-play | Every play with 300+ columns (yards, EPA, WPA, etc.) |
| `extract_player_data()` | Player roster | Names, positions, team history |
| `extract_team_data()` | Teams | Abbreviations, names, colors |
| `extract_schedule_data(seasons)` | Schedule | Game dates, scores, results |

### 2. Transform Players (`transform_players.R`)

`transform_player_stats(pbp_data, season)` → DataFrame with ~40 metrics per player-week

**Passing Stats** (from QB dropbacks):
- `pass_attempts`, `pass_completions`, `pass_yards`, `pass_touchdowns`, `pass_ints`
- `pass_sacks`, `pass_sack_yards`, `pass_qb_hits`
- `pass_air_yards`, `pass_yac_total`, `pass_first_downs`, `pass_20_plus`, `pass_long`
- `pass_epa`, `pass_wpa`, `pass_success_total`, `pass_cpoe_total`

**Rushing Stats** (from rush plays):
- `rush_attempts`, `rush_yards`, `rush_touchdowns`, `rush_long`
- `rush_stuffs`, `rush_10_plus`, `rush_20_plus`, `rush_first_downs`
- `rush_epa_total`, `rush_wpa_total`, `rush_success_total`
- `qb_scramble_*` variants for QB scrambles

**Receiving Stats** (from targets/receptions):
- `rec_targets`, `rec_receptions`, `rec_yards`, `rec_touchdowns`, `rec_long`
- `rec_yac_total`, `rec_first_downs`, `rec_20_plus`, `rec_air_yards_total`
- `rec_epa_total`, `rec_wpa_total`, `rec_success_total`

**Total/Scrimmage Stats** (aggregated):
- `total_yards`, `total_plays`, `total_touchdowns`, `total_epa`, `total_wpa`
- `scrim_yards`, `scrim_touches`, `scrim_touchdowns` (rush + rec only)

### 3. Transform Teams (`transform_teams.R`)

`transform_team_stats(pbp_data, season)` → DataFrame with ~60 metrics per team-week

**Offensive Stats** (`off_*`):
- Pass: `off_pass_attempts`, `off_pass_yards`, `off_pass_touchdowns`, `off_pass_epa`, etc.
- Rush: `off_rush_attempts`, `off_rush_yards`, `off_rush_stuffs`, `off_rush_10_plus`, etc.
- Situational: `off_third_down_attempts/conversions`, `off_fourth_down_*`
- Down splits: `off_early_down_*` (1st/2nd), `off_late_down_*` (3rd/4th)
- Other: `off_turnovers`, `off_fumbles`, `off_three_and_outs`, `off_drives_total`, `off_points_total`

**Defensive Stats** (`def_*`):
- Mirror of offensive stats from opponent's perspective
- Includes: `def_pass_*`, `def_rush_*`, `def_third_down_*`, `def_points_total`, etc.

### 4. Enhance (`utils.R`)

Adds metadata not available in play-by-play data:

| Function | Adds |
|----------|------|
| `enhance_player_data()` | `team_id`, `position` from roster; `win/loss/tie` from schedule |
| `enhance_team_data()` | `win/loss/tie` boolean columns from schedule |

### 5. Load (`load.R`)

Inserts into PostgreSQL tables:

| Function | Table | Data |
|----------|-------|------|
| `load_team_data()` | `teams` | team_id, abbr, display_name, nickname, primary_color |
| `load_player_data()` | `players` | player_id, first_name, last_name, first_season, last_season |
| `load_weeks_data()` | `weeks` | season, week, season_type (REG/POST) |
| `load_player_stats()` | `player_week` | All player weekly statistics |
| `load_team_stats()` | `team_week` | All team weekly statistics |
| `clear_season_data()` | - | Deletes existing data for a season |

## Database Schema

```sql
teams (team_id PK, abbr, abbr3, display_name, nickname, primary_color)
players (player_id PK, first_name, last_name, first_season, last_season)
weeks (season, week, season_type)

player_week (
  player_id FK, season, week, team_id FK, position,
  win, loss, tie,
  pass_*, rush_*, rec_*, total_*, scrim_*  -- ~40 stat columns
)

team_week (
  team_id FK, season, week,
  win, loss, tie,
  off_*, def_*  -- ~60 stat columns
)
```

## Key Implementation Details

### Data Filtering
- **Two-point attempts excluded**: `two_point_attempt != 1 | is.na(two_point_attempt)`
- **Regular season only**: `season_type == "REG"` for most stats
- **Deduplicated plays**: `distinct(game_id, play_id)` prevents double-counting

### Player-Team Mapping
Players are mapped to teams per-week using the play-by-play data itself (which team they appeared on), with roster data as fallback. This handles mid-season trades correctly.

### Stat Calculations Match nflverse Reference
All calculations match nflverse's official implementations:
- EPA/WPA use the play-level values from nflreadr
- Pass stats exclude scrambles (`qb_scramble != 1`)
- Rush stats include lateral yards where applicable

### Error Handling
- `retry_operation()`: Exponential backoff (1s, 2s, 4s) for network failures
- All extract/transform/load functions return `NULL` on failure
- Pipeline continues to next season if one fails

### Performance
- Vectorized dplyr operations (no per-player/per-team loops)
- Single-pass filtering where possible
- Progress tracking for long operations
